# MLab 实施计划

## 目标

从空工作区交付可直接运行的 AI 工具与导航平台：前端 Vue 3，后端 FastAPI。用户配置一个 OpenAI-compatible 模型 API Key 后，可以完成真实流式对话，并使用助手、任务、文稿、资源、工具导航、记忆和设置功能。

详细产品与架构基线见 `PROJECT_PLAN.md`。本文件用于约束实际实施顺序和完成证据。

## 范围

### 必须交付

1. 单仓工程、Docker Compose、环境变量示例和数据库迁移。
2. 注册、登录、刷新 Token、个人资料与数据隔离。
3. 模型供应商凭据、默认模型、连接测试和密钥加密。
4. 会话、消息、SSE 流式输出、停止生成、重新生成。
5. 助手创建/安装、助手市场与对话绑定。
6. 工具库搜索、分类、收藏、外链与可调用工具区分。
7. 任务、文稿、资源和记忆的持久化操作闭环。
8. 响应式 Vue 界面，覆盖参考图中的首页、市场、工具和设置。
9. 后端测试、前端构建和关键端到端流程验证。
10. README、部署说明、故障排查和 `walkthrough.md`。

### 明确边界

- Midjourney、Suno 等独立产品没有凭据时只作为外部导航，不伪造 API 调用。
- 首期预留订阅领域和 UI，不接真实支付渠道。
- 本地可用 SQLite 降低启动门槛；Docker/生产默认 PostgreSQL。

## 实施顺序

### 阶段 A：工程基线

- 创建前后端目录和构建配置。
- 建立统一配置、日志、异常、数据库 Session 与健康检查。
- 配置 Docker Compose、`.env.example`、格式化与检查工具。

验证：前后端依赖可安装；后端健康检查通过；前端能构建。

### 阶段 B：身份与模型设置

- 实现用户、Refresh Token、Provider Credential、Model Config。
- 实现 Argon2 密码、JWT、凭据加密、掩码返回。
- 完成认证和设置 API 以及前端状态管理。

验证：注册登录、Token 轮换、用户隔离、凭据不明文返回。

### 阶段 C：真实聊天闭环

- 实现会话、消息、统一 Provider 和 SSE 协议。
- 支持历史、多轮上下文、停止、重试和自动标题。
- 完成首页、会话页、Composer 和消息渲染。

验证：Fake Provider 自动测试与真实兼容端点手工验证路径都成立。

### 阶段 D：业务模块

- 助手、工具、任务、文稿、资源、记忆数据与 API。
- 对应 Vue 页面、搜索、筛选、编辑和反馈状态。
- 加入系统种子数据与助手/任务模板。

验证：各模块均存在 CRUD 或安装/收藏闭环，不保留纯静态按钮。

### 阶段 E：质量与交付

- 补齐自动化测试、响应式与可访问性检查。
- 执行构建、Lint、测试、迁移和 Docker 验证。
- 更新 README、任务清单与 walkthrough。

验证：按验收矩阵逐项采集当前代码、测试输出和运行截图作为证据。

## 关键技术决策

- 使用模块化单体，降低中小企业部署和事务管理成本。
- 使用 Vanilla CSS 和 CSS Variables 精确还原视觉，符合项目规则。
- 使用 Web 标准 `fetch` 读取 SSE POST 响应，以便携带 JSON 消息请求。
- Provider 统一抽象为异步事件流，首期实现 OpenAI-compatible 适配器。
- API Token 放在 Authorization Header；Refresh Token 首期使用请求体，生产部署文档提供 HttpOnly Cookie 加固方式。
- 对密钥使用应用级对称加密，主密钥仅由环境变量提供。

## 验收方法

- 后端：pytest 覆盖认证、用户隔离、业务 CRUD、SSE 与错误协议。
- 前端：Vitest 覆盖 Store/组件，`vue-tsc` 与 Vite build 验证类型和构建。
- 集成：启动 API 和前端，Playwright 验证桌面/移动视口关键流程。
- 部署：验证 Compose 配置、迁移、健康检查和全新环境说明。
